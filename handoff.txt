# ============================================================
#  WHAT THIS PROJECT IS
# ============================================================

An automated YouTube video production pipeline built in Python.
A single topic prompt (or a pre-written script) produces a fully
narrated, illustrated, captioned MP4 video — completely unattended.
Supports YouTube long-form and Shorts.

T is the project manager / infrastructure guy. Claude is the developer.
T has a friend who also uses the pipeline remotely via Tailscale.

Stack:
  - Ollama (Qwen 2.5 14B) — image prompt generation only
  - Claude/ChatGPT (external) — preferred script writing
  - SearXNG (Docker, port 8080) — local web search for research
  - Chatterbox TTS (ComfyUI workflow) — voice cloning
  - Faster-Whisper (medium, CUDA) — transcription + timestamps
  - Qwen Image (ComfyUI workflow, Wan2.1) — image generation
  - ffmpeg — final video assembly with Ken Burns + captions
  - FastAPI + uvicorn — web server frontend
  - Tailscale — remote access for friend

Hardware: Razer Blade 16, RTX 4090 mobile, Windows 11, Python 3.10

# ============================================================
#  PIPELINE FLOW
# ============================================================

run_pipeline() params:
  topic, auto, mode, style, voice_id, captions,
  caption_words, caption_size, caption_position,
  style_notes, script

1. research.py   — SearXNG generates 8 queries, fetches results,
                   LLM synthesises into a research brief.
                   SKIPPED if script= is provided.

2. llm.py        — LLM writes script grounded in research.
                   SKIPPED if script= is provided.
                   Then generates one image prompt per sentence (always runs).

3. tts.py        — Script split into ~1000-char chunks. Each chunk sent
                   to ComfyUI Chatterbox workflow with selected voice file.
                   Chunks stitched with ffmpeg. VRAM flushed after.

4. transcribe.py — Faster-Whisper transcribes audio with word timestamps.
                   Matches to ORIGINAL script sentences (not Whisper splits).
                   Saves timestamps.json (sentence) and words.json (word-level).
                   VRAM flushed after.

5. imagegen.py   — One image per prompt via ComfyUI Qwen workflow.
                   VRAM flushed after ALL images done (not between).

6. assemble.py   — ffmpeg builds final video:
                   - Ken Burns zoompan on each image (ZOOM_END=1.1)
                   - Images scaled 2x before zoom for headroom
                   - Segment durations from Whisper timestamps
                   - Duo captions burned in for Shorts (if enabled)
                   - Audio: AAC 192k
                   - Output: 1280x720 (long) or 720x1280 (short)

State file: pipeline_state.json is CLEARED at the start of every new
run (not at the end). This was a deliberate fix to prevent previous
video's state bleeding into next topic in batch/server mode.


# ============================================================
#  SCRIPT INPUT MODES
# ============================================================

The pipeline supports two input modes:

PROMPT MODE (default):
  - Topic string → research → local LLM writes script
  - Good for batch runs, experimentation
  - Research quality degrades when SearXNG rate-limits

SCRIPT MODE (preferred for quality):
  - T writes or generates script externally (Claude.ai, ChatGPT)
  - Pastes into the web UI "📄 SCRIPT" tab or passes via --script flag
  - Pipeline uses script verbatim — no rewriting, no research
  - Local LLM only handles image prompt generation (simpler task)
  - Queue items show a purple 📄 SCRIPT badge

The local Qwen 2.5 14B is used ONLY for image prompts in script mode.
Script quality is noticeably better when written by Claude/GPT.

# ============================================================
#  COMFYUI PROCESS MANAGER (comfyui.py)
# ============================================================

comfyui.py owns the ComfyUI process:
  comfyui.start()   — start if not running, wait until ready
  comfyui.stop()    — terminate gracefully
  comfyui.restart() — stop + start (used between batch videos)
  comfyui.is_running() — health check via /system_stats

server.py starts ComfyUI automatically on startup.
batch.py restarts ComfyUI every COMFYUI_RESTART_EVERY videos.
queue_worker in server.py checks ComfyUI health before each job
and auto-restarts if it has crashed.

ComfyUI launch command: python main.py --listen
Uses CREATE_NEW_PROCESS_GROUP on Windows for clean termination.
Readiness detected by polling GET /system_stats (not fixed sleep).


# ============================================================
#  CAPTIONS
# ============================================================

Shorts only. Burns in word-level captions using ffmpeg drawtext.

Technical approach:
  - words.json saved by transcribe.py (word-level timestamps)
  - _build_duo_captions() in assemble.py groups words into chunks
  - Each chunk = one drawtext filter with enable='between(t,start,end)'
  - All drawtext filters comma-chained after the concat filter
  - Filter graph: [kb0][kb1]..concat[precaption],drawtext,...[outv]
  - Written to filter_complex.txt to avoid command-line length limits

Font path escaping (confirmed working on this Windows ffmpeg build):
  C:\Windows\Fonts\arialbd.ttf → C\\:/Windows/Fonts/arialbd.ttf
  Computed in assemble.py: FONT_PATH_ESCAPED

Unicode sanitization:
  - Curly quotes, em-dashes, accented chars → ASCII equivalents
  - Apostrophes dropped entirely (safer than escaping in filter strings)
  - Colons escaped as \:
  - Non-ASCII stripped

Configurable per video:
  caption_words    : 1, 2, or 3 words per chunk
  caption_size     : small (52px) / medium (72px) / large (96px)
  caption_position : top (10%) / middle (45%) / bottom (75%)

   docker cp back + docker restart searxng.
   Note: research.py query prompt still says "documentary video" —
   should be updated to "YouTube video" for funny-style topics.

2. COMFYUI MEMORY CRASHES
   Cause: VRAM accumulation over many videos.
   Fix applied: explicit /free calls after TTS and image gen,
   torch.cuda.empty_cache() after Whisper, ComfyUI restart every 5 videos.
   Status: resolved — 40+ video overnight runs with zero crashes confirmed.

3. CAPTION FILTER GRAPH
   The [precaption] vs [outv] label on the concat filter is decided
   BEFORE _build_filter_complex() is called. This is intentional —
   the earlier approach of string-replacing the filter was fragile.
   If captions fail, check: does words.json exist? Is it non-empty?

4. FONT PATH ESCAPING
   ffmpeg on Windows requires: C\\:/Windows/Fonts/arialbd.ttf
   No surrounding quotes. Computed from CAPTION_FONT in config.
   Tested and confirmed working with ffmpeg 8.1 full build.

T wants the pipeline to upload finished videos to YouTube automatically.

Requirements:
  - Upload after pipeline completes (optional toggle per video)
  - Support multiple YouTube channels (T has 2 channels on one email,
    friend has his own channel)
  - Schedule uploads (not publish immediately) — YouTube supports
    setting publishAt timestamp via the API
  - Default schedule: 3 slots per day. Suggested times (better than
    original 12am/5am/noon): 7:00am, 12:00pm, 6:00pm
  - Remember which slots are already booked (JSON file on disk)
  - Channel selector in the UI (index.html) with other settings
  - OAuth integrated into the web UI (not a separate setup script)
  - Upload toggle in UI (upload or skip)

Technical notes:
  - YouTube Data API v3
  - OAuth2 required per channel — each channel = one token file on disk
  - Initial OAuth requires a browser redirect (one-time per channel)
  - Tokens auto-refresh using refresh_token (long-lived)
  - Upload = 1,600 quota units. Daily quota = 10,000 units (~6 uploads/day)
  - Upload as "private" + publishAt timestamp for scheduling
  - Google Cloud project needed — T does NOT have one yet
  - T wants OAuth setup integrated into the web UI

Suggested build order:
  Phase 1: Google Cloud setup instructions + youtube.py module
           (single channel, manual trigger, no scheduling)
  Phase 2: Multi-channel support (one token file per channel)
  Phase 3: Slot scheduling logic + slot tracker JSON
  Phase 4: UI integration (channel selector, upload toggle, schedule display)

Google Cloud setup T needs to do (one-time, before any code runs):
  1. Go to console.cloud.google.com
  2. Create a new project
  3. Enable "YouTube Data API v3"
  4. Create OAuth 2.0 credentials (Desktop App type)
  5. Add T's Google account as a test user (OAuth consent screen)
  6. Download client_secret.json → put in videoGenerator/

Required pip package: google-auth-oauthlib google-api-python-client


# ============================================================
#  TONE & STYLE NOTES
# ============================================================

SERIOUS style:
  - Authoritative but conversational documentary tone
  - Works for: history, psychology, science, true crime, biography
  - "True crime documentary meets Ken Burns"
  - Opens in the middle of action, no setup
  - Forbidden phrases: "what most people don't know",
    "who's really pulling the strings", "the real story",
    "what nobody talks about", "but here's the thing"

FUNNY style:
  - Confident, irreverent, upbeat YouTube commentary
  - Smart observational humour, not mean-spirited
  - Specificity is funnier than generality
  - Timing lives in sentence structure (short after long = punchline)

Image prompts (both styles):
  - NO TEXT of any kind (signs, letters, storefronts)
  - No named real people, no named real locations
  - No copyrighted material
  - Serious: cinematographer mindset, period accuracy, emotional register
  - Funny: comedy director mindset, deadpan or absurd framing

SCRIPT CONVENTIONS (important for TTS):
  - Write all numbers as words ("nineteen sixty-two" not "1962")
  - Each sentence = one complete thought = one image
  - No section headers, no bullet points
  - No embedded references or citations
